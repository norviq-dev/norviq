## Known Limitations / Workarounds

- Monaco can't mount in jsdom; PolicyCatalog.test.tsx mocks @monaco-editor/react. The e2e (ui-validate)
  asserts the real `.monaco-editor` mounts in a browser.
- AuditLog poll test uses fake timers + advanceTimersByTimeAsync to drive the 5s interval.
- ui-validate is a headed-Chromium run against the live dev servers; run it ONCE (do not per-test rerun)
  and diff against the committed results.json. It needs the API + policy seeded + attack_paths computed
  (for the Simulate path to exist) — see Phase 1 notes.
