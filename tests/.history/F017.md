## Known Limitations / Workarounds

- Integration tests require a reachable local API (uvicorn on 127.0.0.1:8080) + Postgres/Redis from
  docker-compose.dev.yml, and `NRVQ_API_TOKEN` set (or the default-secret token). They skip-with-reason if the API is down (P-11 visible-skip), so run them with the API up.
- The `/ws/audit` broadcast is in-process (single API worker). Multi-worker deployments need a shared
  pub/sub (Redis) for fan-out across workers — not yet implemented.
- Attack baseline drifts with Redis state: clear `agent_history:* agent_profile:* trust:* eval:* agent_class:* agent_frozen:* trustcalc:*` and restart the API before measuring (test-baseline-discipline.md). A polluted run shows safe-allow guards escalating / blocks missing.
- PRE-EXISTING (out of F017 scope, NOT introduced here): `make test` has 27 failures on this env from
  asyncpg `sslmode` kwarg version incompatibility (test fixtures building their own engine), `.env.local`
  overriding config defaults (test_config), evaluator-fixture policy-not-loaded (test_evaluator), missing
  `TrustScore.after_violation` (test_trust/interceptor), and tests not sending auth (test_policy_lifecycle).
  These live in untouched files (evaluator/interceptor/trust/sidecar/telemetry/config/policy_loader).
