# S003 Code Registry

## 1) Feature
- ID: S003
- Name: Exception Classes
- Scope: unified Norviq exception hierarchy with codes and payloads

## 2) Files
- `norviq/exceptions.py`
- `tests/test_exceptions.py`
- `architecture/S003.mmd`
- `registry/S003.md`

## 3) Structure
- `_reason_or_default` safely extracts decision reasons at `norviq/exceptions.py:5`.
- `NorviqError` is the common base carrying `message` and `code` at `norviq/exceptions.py:11`.
- `NorviqBlockError` carries the full `decision` object at `norviq/exceptions.py:20`.
- `NorviqEscalateError` carries the full `decision` object at `norviq/exceptions.py:31`.
- `NorviqTimeoutError` stores `timeout_ms` at `norviq/exceptions.py:42`.
- `NorviqConfigError` stores `field` at `norviq/exceptions.py:50`.

## 4) Methods
- `_reason_or_default(decision, default)` extracts `decision.reason` safely.
- Each exception constructor assigns payload fields and formats `[CODE] message`.
- Falsy or missing decision reasons fall back to default message text.

## 5) Config
- No runtime config reads are required in this feature.

## 6) Data Flow
- Policy/config/runtime failures -> specific `Norviq*Error` instance -> caller catches by base or concrete type.

## 7) Dependencies
- Python `Exception` base class and typed constructor arguments.

## 9) Upstream Table
- F006 raises `NorviqError`.
- F009 raises `NorviqBlockError` and `NorviqEscalateError`.
- F011/F012 catch block and escalate errors.
- F015 catches all exception types via `NorviqError`.

## 10) Error Code Map
- `NorviqError` -> `NRVQ-SDK-1000` (`norviq/exceptions.py:14`)
- `NorviqBlockError` -> `NRVQ-ENG-2010` (`norviq/exceptions.py:23`)
- `NorviqEscalateError` -> `NRVQ-ENG-2015` (`norviq/exceptions.py:34`)
- `NorviqTimeoutError` -> `NRVQ-ENG-2002` (`norviq/exceptions.py:45`)
- `NorviqConfigError` -> `NRVQ-SDK-1002` (`norviq/exceptions.py:53`)

## 11) Concurrency
- No shared mutable state; exception instances are per-raise and immutable after construction.
- No locks, Redis, or DB interactions in this feature.

## 12) Debug Guide
- Error case: catch `NorviqError`, inspect `.code` and `.message`; `str(err)` is `[CODE] message`.
- Timeout case: `NorviqTimeoutError.timeout_ms` holds elapsed milliseconds with code `NRVQ-ENG-2002`.
- Fallback case: if `decision.reason` is missing, block/escalate use default text via `_reason_or_default`.
